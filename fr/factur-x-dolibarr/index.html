Factur-X avec Dolibarr : être prêt pour septembre 2026

Dolibarr sait produire des factures Factur-X. Mais générer un fichier n'est pas la même chose que passer les contrôles officiels. Un Factur-X « qui a l'air correct » peut être rejeté par la plateforme agréée sur une règle EN 16931 ou une règle française. Voici ce que Dolibarr fait déjà, et comment vérifier vos fichiers avant l'échéance.

Ce que Dolibarr gère déjà

Depuis la version 17, Dolibarr génère des factures au format Factur-X : un PDF/A-3 (norme ISO 19005-3) avec un XML CII EN 16931 embarqué. Les versions 18 et 19 (la branche stable en 2026) ont amélioré la conformité au profil EN 16931. Un module de facturation électronique complémentaire, qui gère le raccordement à une plateforme agréée, est en finalisation côté communauté Dolibarr (disponibilité annoncée autour du 3e trimestre 2026).

Rappel du calendrier : au 1er septembre 2026, toutes les entreprises doivent pouvoir recevoir une facture électronique, et les grandes entreprises et ETI doivent l'émettre. Les TPE/PME émettent à partir du 1er septembre 2027. Recevoir concerne donc tout le monde dès 2026.

Pourquoi valider quand même

Dans le modèle à 5 coins, votre facture transite par une plateforme agréée qui vérifie sa conformité. Les rejets les plus fréquents ne viennent pas du PDF mais des données : ~200 règles métier EN 16931 (BR-*, BR-CO-*, BR-S-*) plus le référentiel français vérifient la cohérence des totaux, la ventilation de TVA par catégorie, les arrondis ligne à ligne et les mentions obligatoires. À partir de septembre 2026, quatre champs deviennent obligatoires pour les grandes entreprises et ETI : le SIREN du client, l'adresse de livraison quand elle diffère de la facturation, le type d'opération (biens, services ou les deux) et l'option TVA sur les débits le cas échéant.

Valider un fichier Dolibarr en 10 secondes

Exportez une facture Factur-X depuis Dolibarr, puis déposez le PDF (ou le XML CII) dans le validateur gratuit. Vous obtenez la liste exacte des règles en échec, avec le libellé de chaque erreur : XSD officiel, Schematron EN 16931 complet et référentiel français 2026. Sans inscription, aucun fichier conservé. Le validateur fonctionne aussi sur les factures que vous recevez, utile pour tester votre chaîne de réception dès 2026.

Automatiser la vérification

La même vérification existe en API (POST /v1/validate, option check: "fr-ctc" pour les règles françaises). Pratique si vous générez du Factur-X par lots ou dans un script autour de Dolibarr.

POST /v1/validate
{ "file_base64": "<votre facture Dolibarr en base64>", "check": "fr-ctc" }
