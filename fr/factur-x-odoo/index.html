Factur-X avec Odoo : être prêt pour septembre 2026

Odoo produit du Factur-X nativement avec la localisation française. C'est un bon point de départ, mais chaque facture doit passer les contrôles de la plateforme agréée. Voici ce qu'Odoo gère déjà et comment vérifier vos fichiers avant qu'ils soient rejetés.

Ce qu'Odoo gère déjà

Odoo prend en charge les formats réglementaires (UBL, CII et Factur-X) et intègre un point d'accès Peppol. Le Factur-X est natif dans Odoo 17, 18 et 19 dès que la localisation française est installée : dans Réglages, section « Facturation électronique », activez Factur-X (FR). Le module français de facturation électronique d'Odoo, qui gère le rôle de plateforme agréée, est passé en production début juin 2026 sur les branches 18 et 19.

Rappel du calendrier : au 1er septembre 2026, toutes les entreprises doivent pouvoir recevoir une facture électronique, et les grandes entreprises et ETI doivent l'émettre. Les autres entreprises émettent à partir du 1er septembre 2027.

Pourquoi valider quand même

Le format généré peut être valide sans que les données le soient. Les rejets viennent le plus souvent de ~200 règles métier EN 16931 et du référentiel français : totaux incohérents à cause des arrondis, ventilation de TVA qui ne correspond pas aux lignes, motif d'exonération manquant, ou l'un des quatre nouveaux champs obligatoires de septembre 2026 (SIREN du client, adresse de livraison, type d'opération, option TVA sur les débits). Une configuration Odoo incomplète (compte, taxes, mentions) produit un fichier qui échoue au contrôle.

Valider un fichier Odoo en 10 secondes

Depuis Odoo, téléchargez le PDF Factur-X d'une facture, puis déposez-le dans le validateur gratuit. Vous obtenez la liste exacte des règles en échec (XSD officiel + Schematron EN 16931 + référentiel français 2026), avec le libellé de chaque erreur. Sans inscription, aucun fichier conservé. Le validateur teste aussi les factures reçues, utile pour valider votre chaîne de réception dès 2026.

Automatiser la vérification

La même vérification existe en API (POST /v1/validate, option check: "fr-ctc"). Intégrable dans un flux autour d'Odoo, avant l'envoi vers la plateforme.

POST /v1/validate
{ "file_base64": "<votre facture Odoo en base64>", "check": "fr-ctc" }

Questions fréquentes

Faut-il un module payant pour le Factur-X dans Odoo ?Non : le Factur-X est natif avec la localisation française sur Odoo 17/18/19, sans surcoût d'activation.
Odoo suffit-il pour être conforme ?Odoo peut jouer le rôle de plateforme agréée via son module dédié. La validation en amont reste utile pour éviter les rejets et pour tester les fichiers reçus d'autres éditeurs.
Et si j'utilise une autre PA que celle d'Odoo ?Le validateur est indépendant de la plateforme : il contrôle le fichier lui-même, quelle que soit la PA choisie.
